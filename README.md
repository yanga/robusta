# Brewlytics

A ChatGPT-style data analyst. Upload a CSV or Excel spreadsheet, ask questions in plain English, and get text answers with embedded interactive charts.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React 19, Vite, TypeScript, Tailwind CSS, Zustand, Recharts, Socket.IO client |
| Backend | Express, Socket.IO, PapaParse, SheetJS (xlsx) |
| LLM | Claude API (Anthropic SDK) with streaming + tool use |
| Monorepo | npm workspaces |

## Project Structure

```
├── packages/
│   ├── shared-types/   # TypeScript types shared between frontend & backend
│   ├── backend/        # Express + Socket.IO server, Claude integration
│   └── frontend/       # React + Vite SPA
├── docs/               # Design docs and storyboard
├── .env.example        # Environment variable template
└── package.json        # Root workspace config
```

## Prerequisites

- **Node.js** >= 18 (tested on v22)
- **npm** >= 9
- An **Anthropic API key** ([get one here](https://console.anthropic.com/))

## Getting Started

### 1. Clone the repo

```bash
git clone <repo-url>
cd Robusta
```

### 2. Install dependencies

```bash
npm install
```

This installs dependencies for all three workspace packages.

### 3. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` and add your Anthropic API key:

```
ANTHROPIC_API_KEY=sk-ant-...
PORT=3001
```

### 4. Run in development mode

```bash
npm run dev
```

This starts both servers concurrently:

- **Frontend** — http://localhost:5173 (Vite dev server with HMR)
- **Backend** — http://localhost:3001 (Express + Socket.IO)

The Vite dev server proxies `/api` and WebSocket requests to the backend automatically.

### 5. Build for production

```bash
npm run build
```

Builds all three packages (`shared-types` → `backend` → `frontend`) in order. Frontend output goes to `packages/frontend/dist/`.

## How It Works

1. **Upload** a CSV or Excel file (max 10 MB). The backend parses it with PapaParse/SheetJS, infers column types, and stores the data in memory.
2. **Example questions** are generated by Claude based on the dataset schema and sent to the frontend via Socket.IO.
3. **Ask a question** in the chat input. The backend routes it through Claude with two custom tools:
   - `query_data` — filters, groups, and aggregates the dataset in memory
   - `render_chart` — returns structured chart config (bar, pie, line, area, horizontal bar)
4. **Responses stream** token-by-token over Socket.IO. Charts render inline with Recharts.
5. **Adaptive strategy**: datasets with ≤100 rows use a single-pass approach (full data in context); larger datasets use a two-pass approach (query generation → answer generation).

## Key Features

- Dark / light theme toggle
- Column autocomplete with inline tags in the input bar
- Interactive charts (switch type, fullscreen)
- Visualize any text-only answer with one click
- Multi-session support (upload multiple files, switch between them)
- Resizable sidebar with schema overview and paginated data table
- Fully responsive mobile layout
